# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Static single-page portfolio site for Samuel Rizzon (samuelrizzon.dev). No build system, no framework — just a single `index.html` with inline CSS and JS, plus SEO/AEO support files. Deployed via Vercel (configured in `.vercel/project.json`).

## Architecture

- **`index.html`** — The entire site: markup, styles (CSS custom properties in `:root`), and scripts (bilingual toggle, Himetrica analytics, Lucide icons). Dark theme with `--bg`, `--surface`, `--border`, `--accent` tokens.
- **AEO (Answer Engine Optimization) layer** — Generated by [aeo.js](https://aeojs.org): `llms.txt`, `llms-full.txt`, `ai-index.json`, `docs.json`, `robots.txt`, `sitemap.xml`, `schema.json`. These files help LLMs and search engines understand the site content.
- **`robots.txt`** — Explicitly allows all major AI crawlers (GPTBot, ClaudeBot, PerplexityBot, etc.) and search engines.

## Development

No build step. Open `index.html` in a browser or use any static file server:

```bash
python3 -m http.server 8000
# or
npx serve .
```

## Key Patterns

- **Bilingual support**: EN/PT toggle implemented in inline JS. Both language versions live in the same HTML with `data-lang` attributes.
- **Analytics**: Himetrica tracking events are embedded inline for portfolio interactions.
- **Icons**: Uses Lucide icons via CDN (`unpkg.com/lucide@latest`) plus inline SVGs for social/contact icons.
- **Fonts**: JetBrains Mono (code/headings) and Inter (body) loaded from Google Fonts.
- **No external dependencies to install** — everything is CDN-loaded or inline.

## When Updating Content

If you change portfolio content in `index.html`, the AEO files (`llms.txt`, `llms-full.txt`, `ai-index.json`, `docs.json`) should also be updated to stay in sync. These are the machine-readable versions of the same content.
